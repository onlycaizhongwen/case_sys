# 推荐币子系统实施计划

## 目标

在不改变现有 MA 金叉识别口径的前提下，为脚本增加推荐评分、推荐分层和结构化输出能力。

## 前置条件

- 已完成 requirements 与 design 文档。
- 现有脚本可作为唯一实现入口。
- 本次不新增外部依赖。

## 实施步骤

1. 扩展数据模型。
   - 为 `EnhancedMarketSignal` 增加推荐相关字段和 24h 涨跌幅字段。
   - 新增推荐配置与推荐结果数据结构。
2. 实现推荐引擎。
   - 编写评分函数、等级判断、理由生成和风险摘要逻辑。
   - 提供批量构建推荐列表的方法。
3. 接入主流程与输出。
   - 在 `main()` 中根据有效信号构建推荐结果。
   - 增加控制台摘要、文本报告增强和 JSON 输出。
4. 验证与同步。
   - 进行语法检查。
   - 尝试运行一次本地脚本验证无明显异常。
   - 回写 `status.md` 与任务记录。

## 验证方式

- `python -m py_compile binance_day_contract_realtime_v5.py`
- `python binance_day_contract_realtime_v5.py`
  - 若外部接口返回异常，至少保证脚本可启动且推荐引擎不会因空结果崩溃。

## 风险与回滚

- 若推荐输出影响原有文本报告可读性，优先保留原有信号报告并附加推荐段落。
- 若实时接口不稳定，验证阶段以语法正确和空结果兼容为最低保真标准。

## 检查点

- 推荐逻辑是否与原有 MA 金叉前置条件保持一致。
- 推荐结果是否具备分数、等级、理由、风险摘要。
- JSON 输出是否可被后续系统直接消费。
