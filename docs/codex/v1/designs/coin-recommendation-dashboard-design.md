# 推荐币子系统调度与可视化设计

## 目标

在保持单机、本地、低依赖的前提下，为推荐系统增加常驻服务、定时调度和前端展示能力。

## 方案概览

### 1. 分析入口复用

- 在现有分析脚本中新增 `run_analysis()` 之类的可复用函数。
- CLI 入口继续保留，但核心执行逻辑不再只存在于 `main()`。

### 2. 服务端实现

- 使用 Python 标准库实现轻量 Web 服务，避免引入额外依赖。
- 后台启动调度线程：
  - 服务启动后立即跑一次
  - 之后每 3600 秒执行一次
- 提供接口：
  - `GET /api/status`
  - `GET /api/recommendations`
  - `POST /api/run`

### 3. 前端实现

- 使用静态 HTML/CSS/JS 页面直接请求本地 API。
- 页面展示：
  - 服务状态
  - 最近运行时间
  - 推荐列表
  - 推荐等级分布
  - 手动刷新按钮

## 数据来源

- 服务内存中维护最近一次运行状态。
- 同时复用原有 JSON 文件作为持久化结果。

## 风险与取舍

- 采用标准库 HTTP 服务而不是 Flask，以保证开箱可跑。
- 前端不使用打包链路，控制本次复杂度。
