# 推荐币子系统运行增强设计

## 目标

在不拆分项目结构的前提下，为现有脚本增加“可配置运行”和“结果历史归档”能力。

## 方案概览

### 1. 运行配置模型

- 新增 `RuntimeConfig`，管理候选币数量、K 线天数、输出目录、是否写历史、历史子目录等。
- 通过 `argparse` 接收命令行参数。
- 支持 `--config` 读取 JSON 文件，并覆盖默认值。

### 2. 配置合并策略

- 默认值来自代码内 dataclass。
- 配置文件覆盖默认值。
- 命令行参数优先级最高。

### 3. 输出策略

- 继续写固定文件：
  - `ma_cross_analysis_report.txt`
  - `coin_recommendations.json`
- 额外在 `output/history/{timestamp}/` 下保存同内容快照。
- JSON 增加本次运行的元数据，便于追踪。

## 风险与取舍

- 保持单文件实现，避免本轮增强演变成工程化重构。
- 配置文件先使用 JSON，减少额外依赖。
