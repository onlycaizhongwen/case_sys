# 多周期推荐需求

## 背景

当前推荐系统只有一套日线策略，对应长线候选列表。现在需要扩展为三套策略并行输出：

- 长期：日线 `1d`
- 中期：4 小时线 `4h`
- 短期：1 小时线 `1h`

三者都沿用当前 MA7/MA10 金叉与推荐评分逻辑，但分别基于不同周期独立判断并输出各自候选列表。

## 目标

1. 输出长、中、短三类推荐候选列表。
2. 长线继续使用当前日线逻辑。
3. 中期改为 4 小时 K 线，短期改为 1 小时 K 线。
4. Dashboard 能展示三类候选列表，而不是只展示一类。

## 范围

### In Scope

- 抽象分析周期配置。
- 为三种周期分别执行推荐分析。
- 输出结构扩展为多策略结果。
- 前端支持查看三类候选列表。

### Out of Scope

- 不引入新的技术指标体系。
- 不做不同周期之间的联合打分。
- 不新增数据库。

## 完成标准

1. JSON 输出中存在长/中/短三类结果。
2. 服务接口返回三类候选数据。
3. 页面可明确切换或区分三类候选列表。
4. 现有日线长线逻辑不被破坏。
