# 推荐币子系统运行增强需求

## 背景

当前推荐币子系统已经可以产出推荐列表和 JSON 结果，但运行参数仍写死在脚本中，且每次运行会覆盖固定文件，不利于调参与历史复盘。

## 目标

1. 让核心运行参数可以通过命令行或外部配置文件传入。
2. 保留每次运行的结果快照，便于后续比较不同时间点和不同参数下的推荐结果。
3. 保持当前脚本仍可直接无参运行，不增加部署复杂度。

## 范围

### In Scope

- 增加命令行运行参数。
- 支持读取本地 JSON 配置文件覆盖默认配置。
- 将每次运行结果保存到带时间戳的历史目录。
- 保留最新结果文件，兼容现有使用方式。

### Out of Scope

- 不引入数据库或服务端持久化。
- 不新增调度器。
- 不做回测分析。

## 关键场景

1. 用户可以通过 `--limit`、`--days`、`--output-dir` 等参数控制一次运行。
2. 用户可以通过配置文件调推荐阈值和权重，而不是修改源码。
3. 每次运行后，既有最新文件，也有按时间归档的历史结果。

## 完成标准

1. 脚本支持命令行参数和可选配置文件。
2. 运行结束后会在历史目录写出报告和 JSON 快照。
3. 无参运行兼容旧行为。

## 依赖

- 依赖当前单文件脚本结构。
- 依赖本地文件系统可写。
