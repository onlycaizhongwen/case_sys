# 推荐币子系统需求

## 背景

现有 [binance_day_contract_realtime_v5.py](/d:/Ai办公/case_sys/binance_day_contract_realtime_v5.py) 具备高成交量币种抓取、近 3 日 MA7/MA10 金叉检测、基础风险字段计算与文本报告输出能力，但“检测结果”仍停留在信号列表，缺少独立的推荐层，无法稳定回答“当前应该优先关注哪些币、为什么、风险如何”。

## 目标

1. 在现有分析脚本中补充一个独立的推荐币子系统，把原始信号转换为可排序、可分层、可解释的推荐结果。
2. 推荐结果需要综合金叉新鲜度、趋势强度、置信度、成交量、波动率、RSI 等已有指标，形成统一推荐分数。
3. 输出中需要同时提供推荐等级、推荐理由、风险摘要与建议操作区间，便于直接阅读和后续系统接入。
4. 尽量复用现有脚本的数据抓取与技术指标逻辑，不改变核心 MA 金叉判定口径。

## 范围

### In Scope

- 扩展信号数据结构，使推荐层拥有足够的评分输入。
- 新增推荐配置、推荐结果模型与推荐引擎。
- 将推荐引擎接入现有主流程，对符合条件的币种进行打分、排序、分层。
- 增强输出结果，至少覆盖控制台摘要、文本报告、结构化 JSON 文件。

### Out of Scope

- 不新增数据库、Web 服务、消息队列或定时任务。
- 不改造为多文件工程或引入新的第三方依赖。
- 不重新设计 MA 金叉筛选规则，只在其后增加推荐层。
- 不对接真实下单、仓位管理或自动交易执行。

## 关键场景

1. 运行脚本后，用户可以看到 Top 推荐币列表，而不是只看到全部金叉币列表。
2. 用户可以从推荐结果中直接读到推荐等级、总分、关键理由、风险级别、建议仓位与止盈止损。
3. 后续若需要调参，开发者可以通过集中配置调整评分权重和推荐门槛。

## 完成标准

1. 存在独立的推荐模型和推荐计算逻辑，且不把评分公式硬编码在 `main()` 中。
2. 对每个有效信号都能产出推荐分数、推荐等级和推荐原因。
3. 推荐结果按分数降序输出，并能区分“强烈推荐 / 关注 / 观察”等层级。
4. 至少生成一个 JSON 结果文件，供后续系统消费。
5. 现有脚本仍可直接运行，不破坏原有 MA 金叉分析主流程。

## 非目标

- 不保证推荐收益，不构成投资建议。
- 不做回测系统、参数寻优系统或历史表现评估。

## 依赖

- 依赖 Binance 公共行情接口可访问。
- 依赖现有脚本中的 `PreciseMACrossAnalyzer`、`EnhancedMarketSignal` 和指标计算方法。
