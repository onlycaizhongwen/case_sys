# 推荐币子系统调度与可视化需求

## 背景

当前系统已经可以手动执行推荐分析并产出 JSON/报告，但仍然依赖人工触发，也没有统一的前端可视化入口。

## 目标

1. 增加一个服务进程，可以每 1 小时自动执行一次推荐分析。
2. 提供一个本地可视化 Web 页面，展示最近一次运行状态、推荐列表、关键统计信息。
3. 保持已有推荐算法和输出文件兼容，避免重写核心分析逻辑。

## 范围

### In Scope

- 抽出可复用的分析执行入口，供脚本模式和服务模式共用。
- 新增一个后台服务，内含每小时调度和 HTTP 接口。
- 新增前端静态页面，用于展示最新推荐结果和服务状态。
- 支持手动刷新一次分析任务。

### Out of Scope

- 不引入数据库。
- 不做用户登录、权限控制和远程部署。
- 不做复杂前端工程化构建。

## 关键场景

1. 服务启动后无需人工干预，每小时自动更新一次推荐结果。
2. 用户打开本地页面即可看到最近更新时间、推荐币列表、分层统计和运行状态。
3. 用户可以在页面或接口上手动触发一次立即刷新。

## 完成标准

1. 存在独立的服务启动脚本。
2. 服务启动后会自动执行首轮分析，并继续每小时执行。
3. 页面可正常打开并读取最新结果。
4. 手动刷新接口可用。

## 依赖

- 依赖现有 `binance_day_contract_realtime_v5.py` 分析能力。
- 依赖本地文件系统写入输出文件。
