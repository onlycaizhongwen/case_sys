# 多时间框架一致性加分需求
## 背景

当前推荐系统已经输出长线 `1d`、中期 `4h`、短期 `1h` 三套候选列表，但三套结果仍然是彼此独立评分。
用户希望当同一币种在多个时间框架内同时出现推荐信号时，系统能够识别这种“多周期共振”，并适当提高该币种评分。

## 目标

1. 为同一币种在多个时间框架同时入选推荐时增加一致性加分。
2. 保持现有单周期评分模型主体不变，一致性作为额外 bonus 叠加。
3. 将一致性结果输出到 JSON/API，并在 dashboard 中可见。
4. 排名应随一致性 bonus 调整，使多周期共振币种优先级更高。

## 范围

### In Scope

- 识别同一 `symbol` 在长/中/短三个周期中的同时入选情况
- 按多周期匹配情况计算 bonus
- 输出一致性等级、匹配周期、加分值和摘要说明
- 前端卡片增加一致性信息展示

### Out of Scope

- 不引入新的技术指标体系
- 不引入不同周期之间的复杂加权回归或机器学习模型
- 不改变“先各周期独立算基础分，再统一加一致性 bonus”的主流程

## 规则要求

1. 只有已进入各周期推荐列表的币种才参与一致性判定。
2. 单周期入选不加分。
3. 双周期同时入选时加基础 bonus。
4. 三周期同时入选时加更高 bonus。
5. 若多个周期中至少两个达到 `WATCHLIST` 及以上，再追加小额 bonus。
6. 一致性 bonus 必须有封顶值，避免压过原始评分主体。
7. 最终总分仍需限制在 `0-100` 区间。

## 默认参数

- 双周期一致：`+4`
- 三周期一致：`+8`
- 两个及以上周期达到 `WATCHLIST`：额外 `+2`
- 长线与中期同时成立：额外 `+1`
- 一致性 bonus 上限：`10`

## 输出要求

每条推荐结果至少新增：

- `consistency_level`
- `consistency_bonus`
- `matched_strategies`
- `consistency_summary`

## 完成标准

1. 同一币种在多个周期同时入选时，总分高于未加一致性 bonus 的旧结果。
2. JSON/API 中能看到一致性相关字段。
3. dashboard 卡片能显示多周期共振信息。
4. 单周期币种排序逻辑与原模型基本保持一致，不出现大面积异常翻转。
